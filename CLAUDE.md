## 设计规范

### 视觉参考与玻璃拟态 (Glassmorphism)
复现对象：https://www.raycast.com/  
- **复现深度**：参考 Raycast 的 `Ray-Cast Shadow` 与透明层级。
- **液态玻璃核心参数**：
  - **背景模糊**：`backdrop-filter: blur(20px) saturate(180%);`
  - **背景色**：使用极低透明度的深色 `rgba(17, 18, 19, 0.6)`，禁止使用纯黑或高不透明度灰色。
  - **边框光效**：必须包含 `1px` 的半透明边框 `rgba(255, 255, 255, 0.1)`，用于勾勒轮廓，模拟光线在玻璃边缘的折射。
  - **层级阴影**：使用多层级 `box-shadow`（如：`0 8px 32px 0 rgba(0, 0, 0, 0.8)`），增强悬浮感。
- **透视一致性**：客户端弹出后，背景的 ColorBends 动效必须透过客户端容器清晰可见，形成“悬浮在流光之上的玻璃板”视觉感。

### 组件复现规范 (Ref. Raycast Pricing)
- **导航栏 (Navigation)**：复现其吸顶后的毛玻璃效果与细腻的边框线。
- **选项卡 (Tabs)**：复现 Raycast 风格的胶囊式切换器（Pill Shape），激活项需有微弱的内发光或不同的玻璃折射度。
- **分段容器 (Sections)**：使用 `border-bottom: 1px solid rgba(255, 255, 255, 0.05)` 进行优雅分割。
- **封装要求**：所有 UI 元素（Button, Tab, Card）必须作为独立的 CSS Class 定义，确保在“官网”和“客户端工作台”中调用时视觉高度统一。

### Icon库
统一使用 **Lucide Icons**（CDN引入），禁止混用其他icon库。  
禁止使用emoji替代icon。

### Logo使用
- 路径固定：`./logo.svg`（根目录svg）
- 禁止用容器包裹logo形成app图标效果（如方块包住logo）

## 禁止修改区域

以下内容一旦确定，迭代时不得改动，除非有明确指令：

| 区域 | 约束 |
|------|------|
| 客户端三栏布局比例 | 浏览器内容区70% / AI聊天区30%，不得调整 |
| 液态玻璃效果参数 | backdrop-filter blur值、透明度一旦调好锁定，记录在注释中 |
| Logo引用路径 | 永远是 `./logo.svg`，不得内联或改路径 |
| 导航栏组件 | 复现Raycast导航栏后封装为组件，后续复用，不得重写 |
| Icon库 | 只用Lucide，不得引入其他库 |
| 覆盖层遮罩 | **严禁**在客户端弹出时添加任何不透明或半黑色的 Overlay 遮罩。必须保持背景动效的全量透视。 |

**视觉禁令**：
- 禁止使用 `background-color: #000;` 作为弹窗背景。
- 禁止在客户端下方添加 `opacity: 0.5` 的黑色遮罩层，这会破坏 ColorBends 的色彩饱和度。
- 如果需要突出主体，允许对背景进行整体的 `filter: blur(4px)` 处理，但绝不能遮挡。

## Git 规范 & 部署

每次完成一个有意义的节点必须 commit + push，不需要等整个阶段结束：
- 完成一幕的结构
- 完成一个动画效果
- 修复了一个影响演示的 bug

**commit 格式：** `feat: 第X幕 - [具体内容]` 或 `fix: [问题描述]`

禁止攒多个改动一次 commit。不 commit 直接继续写是被禁止的行为。

**部署工作流（每次 commit 后执行）：**

```
git add index.html
git commit -m "描述"
git push
```

push 完 Vercel 自动部署，约 30 秒后线上更新。

**线上地址：** https://mofang-ai-mvp.vercel.app
